# 4.3. Sơ đồ Sequence

Mục này trình bày các sơ đồ tuần tự (Sequence Diagram) mô tả luồng tương tác giữa các thành phần trong hệ thống thuyết minh tự động phố ẩm thực Vĩnh Khánh. Mỗi sơ đồ tương ứng với một kịch bản sử dụng quan trọng đã được đặc tả trong PRD.

---

## SD-01 — Kích hoạt audio tự động qua GPS / Geofencing

```mermaid
sequenceDiagram
    actor KH as Khách hàng
    participant FE as Frontend
    participant BE as Backend API
    participant DB as Database

    KH->>FE: Bắt đầu tour, cấp quyền GPS
    FE->>FE: watchPosition() — cập nhật mỗi 5 giây
    FE->>FE: Lọc tín hiệu accuracy > 20m, làm mịn MA 3–5 điểm

    FE->>BE: POST /api/location {lat, lng, session_id}
    BE->>DB: SELECT poi_list + cooldown_until
    DB-->>BE: Danh sách POI (lat, lng, radius, priority)

    note over BE: Haversine distance → Priority Queue → chọn POI ưu tiên cao nhất

    alt POI trong geofence và chưa cooldown
        BE->>DB: INSERT location_logs
        BE->>DB: UPDATE cooldown_until = NOW() + cooldown_seconds
        DB-->>BE: OK
        BE-->>FE: POI kích hoạt + audio_url + event ENTER_GEOFENCE
        FE-->>KH: Phát audio thuyết minh tự động (< 2 giây)
        FE->>BE: POST /api/events {type: AUDIO_START, poi_id}
    else Còn trong thời gian cooldown
        BE-->>FE: Không kích hoạt
    end
```

## SD-02 — Quét mã QR

```mermaid
sequenceDiagram
    actor KH as Khách hàng
    participant FE as Frontend
    participant BE as Backend API
    participant DB as Database

    KH->>FE: Mở camera, quét mã QR tại gian hàng
    FE->>FE: Parse deep link → stall_id = 123

    FE->>BE: GET /api/stalls/123?lang={current_lang}
    BE->>DB: SELECT stall + content WHERE id = 123

    alt Gian hàng đang hoạt động
        DB-->>BE: Stall data + audio_url
        BE->>DB: INSERT event {type: QR_SCAN, stall_id: 123}
        note over BE: Ghi đè cooldown_until (bypass GPS)
        BE-->>FE: 200 OK — stall detail + audio_url
        FE-->>KH: Điều hướng trang chi tiết, phát audio ngay lập tức
    else Mã QR không hợp lệ
        BE-->>FE: 404 — "Mã QR không được nhận diện"
        FE-->>KH: Hiển thị thông báo lỗi
    else Gian hàng tạm ngừng
        DB-->>BE: Stall data (inactive)
        BE-->>FE: Stall info + status inactive
        FE-->>KH: "Gian hàng hiện không hoạt động"
    end
```

## SD-03 — Đăng nhập và phân quyền JWT

```mermaid
sequenceDiagram
    actor ND as Người dùng
    participant FE as Frontend
    participant BE as Backend API
    participant DB as Database

    ND->>FE: Nhập email + password
    FE->>BE: POST /auth/login {email, password}
    BE->>DB: SELECT user WHERE email = ?
    DB-->>BE: User record (password_hash, role)

    note over BE: bcrypt.verify(password, hash)

    alt Sai mật khẩu
        BE-->>FE: 401 Unauthorized
        FE-->>ND: Hiển thị thông báo lỗi
    else Đăng nhập thành công
        BE->>BE: generateToken(user_id, role, exp: 24h)
        BE-->>FE: 200 OK + JWT + role
        FE-->>ND: Chuyển hướng đến giao diện theo vai trò
    end

    note over ND,DB: Các request tiếp theo

    ND->>FE: Thao tác cần xác thực
    FE->>BE: Request + Authorization: Bearer {JWT}
    note over BE: Verify JWT signature + expiry
    note over BE: Kiểm tra role vs required_role của endpoint

    alt Đúng quyền
        BE-->>FE: 200 OK + response data
        FE-->>ND: Hiển thị kết quả
    else Sai quyền
        BE-->>FE: 403 Forbidden
        FE-->>ND: Hiển thị thông báo từ chối truy cập
    end
```

## SD-04 — Chủ quán tải nội dung và kiểm duyệt

```mermaid
sequenceDiagram
    actor CQ as Chủ quán
    actor QT as Quản trị viên
    participant FE as Frontend
    participant BE as Backend API
    participant DB as Database

    CQ->>FE: Tải lên file audio/video + chọn ngôn ngữ
    FE->>BE: POST /api/content (multipart: file, stall_id, lang)

    note over BE: Kiểm tra format (mp3/mp4) + size (≤ 10MB)

    alt File không hợp lệ
        BE-->>FE: 400 Bad Request + lý do lỗi
        FE-->>CQ: Hiển thị thông báo lỗi
    else File hợp lệ
        BE->>DB: Upload file, INSERT content {status: PENDING}
        DB-->>BE: OK
        BE-->>FE: 201 Created
        FE-->>CQ: "Nội dung đang chờ duyệt"
    end

    note over QT,DB: Quản trị viên nhận thông báo, vào trang kiểm duyệt

    QT->>FE: Xem danh sách nội dung chờ duyệt
    FE->>BE: GET /api/content/pending
    BE->>DB: SELECT content WHERE status = 'PENDING'
    DB-->>BE: Danh sách content chờ duyệt
    BE-->>FE: 200 OK
    FE-->>QT: Hiển thị danh sách + nghe thử audio

    alt Phê duyệt
        QT->>FE: Nhấn "Phê duyệt"
        FE->>BE: PATCH /api/content/{id}/approve
        BE->>DB: UPDATE status = APPROVED
        DB-->>BE: OK
        BE-->>FE: 200 OK
        FE-->>QT: Cập nhật trạng thái
        BE-->>CQ: Thông báo nội dung đã được duyệt
    else Từ chối
        QT->>FE: Nhấn "Từ chối" + nhập lý do
        FE->>BE: PATCH /api/content/{id}/reject {reason}
        BE->>DB: UPDATE status = REJECTED
        DB-->>BE: OK
        BE-->>FE: 200 OK
        FE-->>QT: Cập nhật trạng thái
        BE-->>CQ: Thông báo từ chối kèm lý do
    end
```

## SD-05 — Bắt đầu tour và hiển thị bản đồ

```mermaid
sequenceDiagram
    actor KH as Khách hàng
    participant FE as Frontend
    participant BE as Backend API
    participant DB as Database

    KH->>FE: Nhấn "Bắt đầu tour" + nhập ngân sách (tuỳ chọn)
    FE->>FE: Yêu cầu quyền Geolocation

    alt Từ chối GPS
        FE-->>KH: Dialog yêu cầu cấp quyền, tour chưa bắt đầu
    else Cho phép GPS
        FE->>BE: POST /api/sessions {budget, lang, lat, lng}
        BE->>DB: INSERT visit_session {budget, lang, started_at}
        DB-->>BE: session_id
        BE->>DB: SELECT active POI list
        DB-->>BE: Danh sách POI (lat, lng, radius, priority)
        BE-->>FE: session_id + POI list
        FE->>FE: Khởi động watchPosition() mỗi 5 giây
        FE-->>KH: Bản đồ + marker gian hàng + vị trí người dùng
    end

    alt Bỏ qua nhập ngân sách
        note over FE: Tour bắt đầu, ẩn tính năng theo dõi chi tiêu
    end
```

## SD-07 — Theo dõi chi tiêu và đổi voucher

```mermaid
sequenceDiagram
    actor KH as Khách hàng
    participant FE as Frontend
    participant BE as Backend API
    participant DB as Database

    note over KH,DB: Phần 1 — Ghi nhận chi tiêu

    KH->>FE: Nhập số tiền chi tiêu tại gian hàng
    FE->>BE: POST /api/expenses {session_id, stall_id, amount}
    BE->>DB: INSERT expense + UPDATE session.budget_remaining
    DB-->>BE: budget_remaining mới
    BE-->>FE: 200 OK + remaining_budget
    FE-->>KH: Cập nhật thanh ngân sách còn lại

    note over KH,DB: Phần 2 — Đổi voucher

    KH->>FE: Nhập mã voucher
    FE->>BE: POST /api/vouchers/redeem {code, stall_id, session_id}
    BE->>DB: SELECT voucher WHERE code = ? AND stall_id = ?

    note over BE: Kiểm tra: còn hạn, chưa dùng, đúng gian hàng

    alt Voucher hợp lệ
        BE->>DB: UPDATE voucher SET used = true
        DB-->>BE: discount_amount
        BE-->>FE: 200 OK + discount_amount
        FE-->>KH: Hiển thị ưu đãi, cập nhật chi tiêu
    else Voucher hết hạn hoặc không hợp lệ
        BE-->>FE: 400 — "Voucher đã hết hạn hoặc không hợp lệ"
        FE-->>KH: Hiển thị thông báo lỗi
    end
```

## SD-08 — Quản trị viên thêm và cấu hình POI

```mermaid
sequenceDiagram
    actor QT as Quản trị viên
    participant FE as Frontend
    participant BE as Backend API
    participant DB as Database

    QT->>FE: Click vị trí trên bản đồ
    FE-->>QT: Tọa độ tự điền (lat, lng)
    QT->>FE: Nhập tên, bán kính (10–200m), ưu tiên (1–10), cooldown

    FE->>BE: POST /api/pois {name, lat, lng, radius, priority, cooldown}
    note over BE: Validate: radius trong khoảng 10–200m

    alt Radius ngoài khoảng hợp lệ
        BE-->>FE: 400 — "Bán kính hợp lệ từ 10m đến 200m"
        FE-->>QT: Hiển thị thông báo lỗi
    else Hợp lệ
        BE->>DB: INSERT poi + geofence_config
        BE->>DB: INSERT qr_code {deep_link: vinhkhanh://stall/{poi_id}}
        DB-->>BE: OK
        BE-->>FE: 201 Created
        FE-->>QT: Marker mới xuất hiện trên bản đồ
    end
```

## SD-09 — Quản trị viên tạo mã QR hàng loạt

```mermaid
sequenceDiagram
    actor QT as Quản trị viên
    participant FE as Frontend
    participant BE as Backend API
    participant DB as Database

    QT->>FE: Nhấn "Tạo QR hàng loạt"
    FE->>BE: POST /api/qr/bulk-generate
    BE->>DB: SELECT stalls WHERE status = 'active'
    DB-->>BE: Danh sách stall_id

    note over BE: Kiểm tra từng stall đã có QR chưa

    alt Một số stall đã có QR cũ
        BE-->>FE: Cần xác nhận ghi đè
        FE-->>QT: "Ghi đè mã cũ đã in?"
        QT->>FE: Xác nhận tiếp tục

    BE->>DB: UPSERT qr_codes (image_url, updated_at)
    DB-->>BE: OK
    BE-->>FE: Download link (ZIP ảnh / PDF in ấn)
    FE-->>QT: Tải file về
```

## SD-10 — Xem thống kê (chủ quán và quản trị viên)

```mermaid
sequenceDiagram
    actor CQ as Chủ quán
    actor QT as Quản trị viên
    participant FE as Frontend
    participant BE as Backend API
    participant DB as Database

    note over CQ,DB: Luồng A — Chủ quán xem thống kê gian hàng

    CQ->>FE: Nhấn "Thống kê"
    FE->>BE: GET /api/vendor/stats?stall_id={id}&range={from,to}
    BE->>DB: Aggregate visits, audio_completion_rate, vouchers_used
    DB-->>BE: Kết quả tổng hợp theo ngày
    BE-->>FE: 200 OK + stats data
    FE-->>CQ: Lượt ghé, tỷ lệ hoàn thành audio, voucher đã dùng

    alt Chưa có lượt ghé thăm
        BE-->>FE: stats = 0
        FE-->>CQ: Số liệu bằng 0 + hướng dẫn thiết lập nội dung
    end

    note over QT,DB: Luồng B — Admin xem phân tích toàn hệ thống

    QT->>FE: Nhấn "Phân tích hệ thống" + chọn khoảng thời gian
    FE->>BE: GET /api/admin/analytics?range={from,to}
    BE->>DB: Tổng phiên tour, top POI, heatmap, peak hours, lang breakdown
    DB-->>BE: Kết quả phân tích tổng hợp
    BE-->>FE: 200 OK + analytics data
    FE-->>QT: Bản đồ nhiệt POI + giờ cao điểm + tỷ lệ theo ngôn ngữ

    alt Không có dữ liệu trong khoảng thời gian chọn
        BE-->>FE: stats empty
        FE-->>QT: "Chưa có dữ liệu trong khoảng thời gian này"
    end
```
