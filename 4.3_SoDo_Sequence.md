# 4.3. Sơ đồ Sequence

Mục này trình bày các sơ đồ tuần tự (Sequence Diagram) mô tả luồng tương tác giữa các thành phần trong hệ thống thuyết minh tự động phố ẩm thực Vĩnh Khánh. Mỗi sơ đồ tương ứng với một kịch bản sử dụng quan trọng đã được đặc tả trong PRD.

---

## SD-01 — Kích hoạt audio tự động qua GPS / Geofencing

```mermaid
sequenceDiagram
    participant TB as Thiết bị<br/>(Mobile browser)
    participant FE as Frontend<br/>(React / GPS)
    participant BE as Backend API<br/>(Spring Boot)
    participant DB as Database<br/>(MySQL)

    TB->>FE: Geolocation.watchPosition()
    FE-->>TB: coords (lat, lng, accuracy)

    note over FE: accuracy > 20m → bác bỏ tín hiệu

    FE->>FE: Làm mịn dữ liệu (trung bình trượt 3–5 điểm)
    FE->>BE: POST /api/location {lat, lng, session_id}
    BE->>DB: Query POI list + cooldown_until
    DB-->>BE: Danh sách POI (lat, lng, radius, priority)

    note over BE: Tính Haversine → Priority Queue → chọn POI ưu tiên cao nhất

    BE->>DB: INSERT location_logs
    BE->>DB: UPDATE cooldown_until = NOW() + cooldown_seconds
    DB-->>BE: OK

    alt POI trong geofence và chưa cooldown
        BE-->>FE: POI kích hoạt + audio_url + event ENTER_GEOFENCE
        FE->>TB: Phát audio thuyết minh (< 2 giây)
        FE->>BE: POST /api/events {type: AUDIO_START, poi_id}
    else Còn trong thời gian cooldown
        BE-->>FE: Không kích hoạt
    end
```

## SD-02 — Quét mã QR

```mermaid
sequenceDiagram
    participant ND as Người dùng<br/>(Mobile browser)
    participant QR as QR Scanner<br/>(Camera API)
    participant BE as Backend API<br/>(Spring Boot)
    participant DB as Database<br/>(MySQL)

    ND->>QR: Mở camera, quét mã QR
    QR-->>ND: Deep link: vinhkhanh://stall/123

    note over ND: Parse stall_id từ deep link

    ND->>BE: GET /api/stalls/123?lang={current_lang}
    BE->>DB: SELECT stall + content WHERE id = 123
    DB-->>BE: Stall data + audio_url theo ngôn ngữ

    alt Gian hàng đang hoạt động
        BE->>DB: INSERT event {type: QR_SCAN, stall_id: 123}
        note over BE: Ghi đè cooldown_until (bypass GPS)
        DB-->>BE: OK
        BE-->>ND: 200 OK — stall detail + audio_url
        ND->>ND: Điều hướng trang chi tiết, phát audio ngay lập tức
    else Mã QR không hợp lệ
        BE-->>ND: 404 — "Mã QR không được nhận diện"
    else Gian hàng tạm ngừng hoạt động
        BE-->>ND: Trang thông tin + "Gian hàng hiện không hoạt động"
    end
```

## SD-03 — Đăng nhập và phân quyền JWT

```mermaid
sequenceDiagram
    participant CL as Client<br/>(Browser)
    participant AC as Auth Controller<br/>(Spring Security)
    participant JW as JWT Service<br/>(Token generator)
    participant DB as Database<br/>(Users table)

    CL->>AC: POST /auth/login {email, password}
    AC->>DB: SELECT user WHERE email = ?
    DB-->>AC: User record (password_hash, role)

    note over AC: bcrypt.verify(password, hash)

    alt Sai mật khẩu
        AC-->>CL: 401 Unauthorized
    else Đăng nhập thành công
        AC->>JW: generateToken(user_id, role)
        JW-->>AC: JWT token (exp: 24h)
        AC-->>CL: 200 OK + JWT + role
    end

    note over CL,AC: Các request tiếp theo

    CL->>AC: Request + Authorization: Bearer {JWT}
    note over AC: Verify JWT signature + expiry
    note over AC: Kiểm tra role vs endpoint required_role

    alt Đúng quyền
        AC-->>CL: 200 OK + response data
    else Sai quyền
        AC-->>CL: 403 Forbidden
    end
```

## SD-04 — Chủ quán tải nội dung và kiểm duyệt

```mermaid
sequenceDiagram
    participant CQ as Chủ quán<br/>(Vendor portal)
    participant BE as Backend API<br/>(Spring Boot)
    participant FS as File Storage<br/>(S3 / local)
    participant QT as Quản trị viên<br/>(Admin portal)

    CQ->>BE: POST /api/content (multipart: file, stall_id, lang)
    note over BE: Kiểm tra format (mp3/mp4) + size (≤ 10MB)

    alt File không hợp lệ
        BE-->>CQ: 400 Bad Request + lý do lỗi
    else File hợp lệ
        BE->>FS: Upload file audio/video
        FS-->>BE: file_url
        BE->>BE: INSERT content {stall_id, lang, url, status=PENDING}
        BE-->>CQ: 201 Created — "Đang chờ duyệt"
    end

    note over QT: Quản trị viên nhận thông báo nội dung mới

    QT->>BE: GET /api/content/pending
    BE-->>QT: Danh sách content chờ duyệt

    alt Phê duyệt
        QT->>BE: PATCH /api/content/{id}/approve
        BE->>BE: UPDATE status = APPROVED
        BE-->>CQ: Thông báo: nội dung đã được duyệt
        BE-->>QT: 200 OK
    else Từ chối
        QT->>BE: PATCH /api/content/{id}/reject {reason}
        BE->>BE: UPDATE status = REJECTED
        BE-->>CQ: Thông báo từ chối + lý do
        BE-->>QT: 200 OK
    end
```

## SD-05 — Bắt đầu tour và hiển thị bản đồ

```mermaid
sequenceDiagram
    participant KH as Khách hàng<br/>(Mobile browser)
    participant FE as Frontend<br/>(React + GPS)
    participant BE as Backend API<br/>(Spring Boot)
    participant DB as Database<br/>(MySQL)

    KH->>FE: Nhấn "Bắt đầu tour" + nhập ngân sách

    note over FE: Yêu cầu quyền Geolocation API

    alt Từ chối GPS
        FE-->>KH: Hiển thị dialog yêu cầu cấp quyền, tour chưa bắt đầu
    else Cho phép GPS
        FE->>BE: POST /api/sessions {budget, lang, lat, lng}
        BE->>DB: INSERT visit_session {budget, lang, started_at}
        DB-->>BE: session_id
        BE->>DB: SELECT active POI list
        DB-->>BE: Danh sách POI (lat, lng, radius, priority)
        BE-->>FE: session_id + POI list
        FE->>FE: Khởi động GPS watchPosition() mỗi 5 giây
        FE-->>KH: Hiển thị bản đồ + marker tất cả gian hàng + vị trí người dùng
    end

    alt Bỏ qua nhập ngân sách
        note over FE: Tour bắt đầu, ẩn tính năng theo dõi chi tiêu
    end
```

## SD-06 — Chuyển đổi ngôn ngữ (Việt / Anh / Hàn)

```mermaid
sequenceDiagram
    participant KH as Khách hàng<br/>(Mobile browser)
    participant FE as Frontend<br/>(i18n / State)
    participant BE as Backend API<br/>(Spring Boot)
    participant DB as Database<br/>(Content table)

    KH->>FE: Chọn ngôn ngữ "한국어" (tiếng Hàn)
    FE->>FE: Lưu lang = "ko" vào localStorage
    FE->>FE: Re-render toàn bộ giao diện sang tiếng Hàn

    FE->>BE: GET /api/stalls/{id}?lang=ko
    BE->>DB: SELECT content WHERE stall_id = {id} AND lang = 'ko'

    alt Có nội dung tiếng Hàn
        DB-->>BE: text_content + audio_url (ko)
        BE-->>FE: Stall content (ko)
        FE->>FE: Phát lại audio tiếng Hàn từ đầu
        FE-->>KH: Giao diện + audio đã chuyển sang tiếng Hàn
    else Chưa có audio tiếng Hàn
        DB-->>BE: text_content only (ko)
        BE-->>FE: Stall content không có audio_url
        FE-->>KH: Hiển thị nội dung văn bản thay thế, không báo lỗi
    end
```

## SD-07 — Theo dõi chi tiêu và đổi voucher

```mermaid
sequenceDiagram
    participant KH as Khách hàng<br/>(Mobile browser)
    participant FE as Frontend<br/>(React)
    participant BE as Backend API<br/>(Spring Boot)
    participant DB as Database<br/>(MySQL)

    note over KH,DB: Phần 1 — Ghi nhận chi tiêu

    KH->>FE: Nhập số tiền chi tiêu tại gian hàng
    FE->>BE: POST /api/expenses {session_id, stall_id, amount}
    BE->>DB: INSERT expense + UPDATE session.budget_remaining
    DB-->>BE: budget_remaining mới
    BE-->>FE: 200 OK + remaining_budget
    FE-->>KH: Cập nhật thanh ngân sách còn lại

    note over KH,DB: Phần 2 — Đổi voucher

    KH->>FE: Nhập mã voucher
    FE->>BE: POST /api/vouchers/redeem {code, stall_id, session_id}
    BE->>DB: SELECT voucher WHERE code = ? AND stall_id = ?

    note over BE: Kiểm tra: còn hạn, chưa dùng, đúng gian hàng

    alt Voucher hợp lệ
        BE->>DB: UPDATE voucher SET used = true
        DB-->>BE: discount_amount
        BE-->>FE: 200 OK + discount_amount
        FE-->>KH: Hiển thị ưu đãi, cập nhật chi tiêu tiết kiệm
    else Voucher hết hạn hoặc đã dùng
        BE-->>FE: 400 — "Voucher đã hết hạn hoặc không hợp lệ"
        FE-->>KH: Hiển thị thông báo lỗi
    end
```

## SD-08 — Quản trị viên thêm và cấu hình POI

```mermaid
sequenceDiagram
    participant QT as Quản trị viên<br/>(Admin portal)
    participant MP as Map Component<br/>(Leaflet/Mapbox)
    participant BE as Backend API<br/>(Spring Boot)
    participant DB as Database<br/>(MySQL)

    QT->>MP: Click vị trí trên bản đồ
    MP-->>QT: Tọa độ tự điền (lat, lng)

    note over QT: Nhập: tên gian hàng, bán kính (10–200m),<br/>mức ưu tiên (1–10), cooldown (300–3600s)

    QT->>BE: POST /api/pois {name, lat, lng, radius, priority, cooldown}

    note over BE: Validate: radius trong khoảng 10–200m

    alt Radius ngoài khoảng hợp lệ
        BE-->>QT: 400 — "Bán kính hợp lệ từ 10m đến 200m"
    else Hợp lệ
        BE->>DB: INSERT poi + geofence_config
        DB-->>BE: poi_id
        BE->>DB: INSERT qr_code {deep_link: vinhkhanh://stall/{poi_id}}
        DB-->>BE: OK
        BE-->>QT: 201 Created
        QT->>MP: Marker mới xuất hiện trên bản đồ
    end
```

## SD-09 — Quản trị viên tạo mã QR hàng loạt

```mermaid
sequenceDiagram
    participant QT as Quản trị viên<br/>(Admin portal)
    participant BE as Backend API<br/>(Spring Boot)
    participant QG as QR Generator<br/>(ZXing / qrcode-js)
    participant DB as Database<br/>(MySQL)

    QT->>BE: POST /api/qr/bulk-generate
    BE->>DB: SELECT stalls WHERE status = 'active'
    DB-->>BE: Danh sách stall_id

    note over BE: Kiểm tra từng stall đã có QR chưa

    alt Một số stall đã có QR cũ
        BE-->>QT: Confirm — "Ghi đè mã cũ đã in?"
        QT->>BE: Xác nhận tiếp tục
    end

    loop Với mỗi stall_id
        BE->>QG: generate(vinhkhanh://stall/{id})
        QG-->>BE: QR image (PNG)
        BE->>DB: UPSERT qr_codes SET image_url, updated_at
    end

    DB-->>BE: OK
    BE-->>QT: Download link (ZIP ảnh / PDF để in ấn)
```

## SD-10 — Xem thống kê (chủ quán và quản trị viên)

```mermaid
sequenceDiagram
    participant CQ as Chủ quán<br/>(Vendor portal)
    participant QT as Quản trị viên<br/>(Admin portal)
    participant BE as Backend API<br/>(Spring Boot)
    participant DB as Database<br/>(Analytics queries)

    note over CQ,DB: Luồng A — Chủ quán xem thống kê gian hàng

    CQ->>BE: GET /api/vendor/stats?stall_id={id}&range={from,to}
    BE->>DB: Aggregate: total_visits, audio_completion_rate, vouchers_used
    DB-->>BE: Kết quả tổng hợp theo ngày (realtime)
    BE-->>CQ: Dashboard: lượt ghé, tỷ lệ hoàn thành audio, voucher đã dùng

    alt Chưa có lượt ghé thăm
        BE-->>CQ: Số liệu = 0 + hướng dẫn thiết lập nội dung
    end

    note over CQ,DB: Luồng B — Admin xem phân tích toàn hệ thống

    QT->>BE: GET /api/admin/analytics?range={from,to}
    BE->>DB: Tổng phiên tour, top POI, heatmap data
    DB-->>BE: Visit heatmap + peak hours
    BE->>DB: Audio completion % phân theo ngôn ngữ
    DB-->>BE: Lang breakdown (vi/en/ko)
    BE-->>QT: Dashboard: bản đồ nhiệt POI + giờ cao điểm + tỷ lệ theo ngôn ngữ

    alt Không có dữ liệu trong khoảng thời gian chọn
        BE-->>QT: "Chưa có dữ liệu trong khoảng thời gian này"
    end
```
