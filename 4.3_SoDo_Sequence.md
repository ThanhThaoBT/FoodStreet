## SD-01 — Bắt đầu phiên tham quan + GPS tracking

```mermaid
sequenceDiagram
    actor KH as Khách hàng
    participant FE as Mobile App
    participant BE as Backend
    participant DB as Database

    KH->>FE: Bắt đầu tour
    FE->>BE: POST /sessions (lat, lng)

    BE->>DB: Insert visit_session
    DB-->>BE: session_id

    BE-->>FE: session_id

    loop mỗi 5s
        FE->>BE: POST /location (lat, lng, session_id)
        BE->>DB: Insert location_logs
        BE-->>FE: OK
    end
```

## SD-02 — Xác định stall gần nhất (trigger logic)

```mermaid
sequenceDiagram
    participant FE as Mobile App
    participant BE as Backend
    participant DB as Database

    FE->>BE: POST /location (lat, lng)

    BE->>DB: Query stalls + trigger_config
    DB-->>BE: Danh sách stalls

    loop từng stall
        BE->>BE: Tính distance
        BE->>BE: Check radius
        BE->>BE: Check cooldown
    end

    BE->>BE: Chọn stall tốt nhất (priority)

    alt Có stall hợp lệ
        BE-->>FE: stallId + trigger
    else Không có
        BE-->>FE: No trigger
    end
```

## SD-03 — Lấy audio (CASE 1: chưa có audio)

```mermaid
sequenceDiagram
    actor KH as Khách hàng
    participant FE as Mobile App
    participant BE as Backend
    participant DB as Database
    participant TTS
    participant Cloud

    KH->>FE: Di chuyển vào vùng stall
    FE->>BE: GET /audio?stallId&lang

    BE->>DB: Query stall_translation
    DB-->>BE: audioStatus = PENDING

    BE->>DB: Update status = PROCESSING

    par Async xử lý
        BE->>TTS: Generate speech
        TTS-->>BE: mp3 file
        BE->>Cloud: Upload
        Cloud-->>BE: audioUrl
        BE->>DB: Update audioUrl, hash, fileSize, COMPLETED
    end

    BE-->>FE: PROCESSING
    FE-->>KH: Đang tải audio...
```

## SD-05 — Cache audio (CASE 3: zero latency)

```mermaid
sequenceDiagram
    participant FE as Mobile App
    participant BE as Backend
    participant DB as Database

    FE->>BE: GET /audio?clientHash=abc123
    BE->>DB: Query translation
    DB-->>BE: audioHash

    alt Hash trùng
        BE-->>FE: needDownload = false

        BE-->>FE: audioUrl
```

## SD-06 — Download hoặc stream audio

```mermaid
sequenceDiagram
    actor KH as Khách hàng
    participant FE as Mobile App
    participant BE as Backend

    FE->>BE: GET /audio
    BE-->>FE: audioUrl + fileSize

    FE->>FE: Check storage

    alt Đủ bộ nhớ
        FE->>BE: Download audio
        FE->>KH: Phát audio từ cache
    else Không đủ
        FE->>BE: Stream audio
        FE->>KH: Phát audio online
    end
```

## SD-09 — Luồng tổng hợp (GPS → Audio)

```mermaid
sequenceDiagram
    actor KH as Khách hàng
    participant FE as Mobile
    participant BE as Backend
    participant DB as Database

    KH->>FE: Di chuyển
```

# 4.4. Activity Diagram — Audio Processing

```mermaid
flowchart TD
    A[Start] --> B{audioStatus}

    B -->|PENDING| C[Set PROCESSING]
    C --> D[Generate audio async]
    D --> E[Return PROCESSING]

    B -->|PROCESSING| F[Return PROCESSING]

    B -->|COMPLETED| G[Compare hash]
    G -->|Match| H[No download]
    G -->|Not match| I[Return audioUrl]
```
